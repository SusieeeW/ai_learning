1. AI的核心概念理解

🧠
基础层：LLM、Token、Context
LLM是大语言模型，Token是AI处理文字的最小单位，Context是上下文
🛠️
工具层：Prompt、Tool、MCP
Prompt是提示词，Tool是我们常用的工具（python、slide等），MCP是工具标准连接协议
🚀
应用层：Agent、Skill
Agent是智能体/自主AI，Skill是技能/标准流程

2.2 AI解决方案

👇 可执行示例

-- 输入需求：
--base: "帮我写个SQL，计算近30天各渠道的新客通过率"
--improve: "根据XX文件中的表结构，帮我写个SQL，计算近30天各渠道的新客通过率"
-- AI 生成结果：
SELECT
channel,
COUNT(DISTINCT user_id) AS apply_cnt,
SUM(CASE WHEN status = 'pass' THEN 1 ELSE 0 END) AS pass_cnt,
ROUND(SUM(CASE WHEN status = 'pass' THEN 1 ELSE 0 END) * 100.0 / COUNT(user_id), 2) AS pass_rate
FROM new_customer_apply
WHERE apply_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY channel;

4. 第三道菜 — 生成一个分享

这次分享是如何用AI做出来的？

1
写分享概要
先和AI沟通，确定分享的主题、目标受众、时长，然后生成一个初步的内容框架。包括：封面、背景、目录、核心概念、应用案例、总结等模块。
2
扩充分享内容，自己修改完善内容稿
AI根据框架生成详细内容后，我们需要自己阅读、修改和完善。比如调整口语化表达、补充专业细节、加入自己的真实案例，让内容更符合我们的实际需求。
3
根据输出，写修改需求，进行完善
PPT初稿出来后，列出具体的修改需求（比如：增加科技感、调整布局、增加实操案例等），让AI继续迭代优化，直到达到满意的效果。
